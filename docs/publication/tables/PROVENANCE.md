# Table Data Provenance

All CSVs in this directory are generated by `extract_tables.py` from the raw
benchmark and profiling data stored in `benchmark_results/` and
`docs/publication/figures/`.  Run the script to regenerate:

```
python3 docs/publication/tables/extract_tables.py
```

---

## table1_hardware.csv

**Paper section:** 6.1  
**Caption:** Test system hardware configuration.

**Source:** Static metadata — transcribed from the test machine at the time of
the experiments (April 2026).  There is no raw log or benchmark file to derive
this from; the rows are defined directly in `HARDWARE_ROWS` inside
`extract_tables.py` and serve as the single source of truth.

**To update:** Edit the `HARDWARE_ROWS` list in `extract_tables.py` and re-run
the script.

## table2_parallel_compression.csv

**Paper section:** 7.2  
**Caption:** Wall-clock time (seconds) with parallel output compression.

**Source file:**  
`benchmark_results/fastp-gpu_v1.2.2/vs_opengene_v1.1.0/galaxy_arm_a100/20260417/full_benchmark_20260417.csv`

**Extraction:**  
Filter columns `dataset`, `threads`, `tool`, `walltime_s`.  Pivot the three
tools (opengene, d0bromir_cpu, d0bromir_gpu) into wide format per
(dataset, threads) key.  Compute `cpu_speedup = opengene_s / d0bromir_cpu_s`
and `gpu_speedup = opengene_s / d0bromir_gpu_s`.

**Run conditions:** gzip-compressed output; default adapter/quality filter
settings; single repetition per cell in the canonical CSV.  Thread counts
1,2,4,8,16 for all datasets; additionally T=32 for Panel_SE_148M,
Panel_PE_304M, WGS_SE_6.3G, WGS_PE_12.8G.

## table5_correctness.csv

**Paper section:** 7.5  
**Caption:** Correctness verification — reads_in / reads_out match OpenGene
baseline at all tested thread counts.

**Source file:**  
`benchmark_results/fastp-gpu_v1.2.2/vs_opengene_v1.1.0/galaxy_arm_a100/20260417/full_benchmark_20260417.csv`

**Extraction:**  
Group all rows by dataset.  Collect the unique set of `reads_in` and
`reads_out` values across all tools and thread counts.  If the set has more
than one element the result field is set to FAIL (all rows should be
identical).  Pass rate = reads_out / reads_in × 100.

**Note:** The benchmark script aborts on any read-count mismatch, so FAIL here
would indicate post-hoc data corruption, not a live benchmark failure.

## table6_kernel_filtering.csv

**Paper section:** 7.3  
**Caption:** Filtering-stage wall time (ms): CPU path vs. GPU kernel.

**Source file:**  
`docs/publication/figures/fig2_kernel_speedup.csv`

**Extraction:**  
Direct copy with column rename: `d0bromir_cpu_filtering_ms` → `cpu_filter_ms`,
`d0bromir_gpu_filtering_ms` → `gpu_filter_ms`.  Speedup = cpu / gpu.

**How fig2_kernel_speedup.csv was originally produced:**  
The `FASTP_PROFILING` framework accumulates `cpu_filter_ms` (time spent in the
filtering stage across all worker threads) and emits it on `[PROF_CSV]` lines.
For the GPU path the same counter captures the time from H2D dispatch to D2H
completion plus CPU-side packing.  Values were extracted from profiling logs
collected during the `run_paper_bench_pe.sh` and `run_paper_bench.sh` runs and
entered into `fig2_kernel_speedup.csv` by hand (per-kernel H2D/D2H/compute
sub-instrumentation returned 0 in the current build).

## table7_og_vs_cpu.csv

**Paper section:** 7.4  
**Caption:** OpenGene v1.1.0 vs. d0bromir CPU wall-clock time (seconds) and
speedup across all datasets and thread counts (gzip-compressed output enabled).

**Source file:**  
`benchmark_results/fastp-gpu_v1.2.2/vs_opengene_v1.1.0/galaxy_arm_a100/20260417/full_benchmark_20260417.csv`

**Extraction:**  
Same pivot as Table 2 but retaining only the opengene and d0bromir_cpu columns.
Speedup = opengene_s / d0bromir_cpu_s.

**Note on T=32 extrapolated rows:**  
The paper includes two dagger-marked (†) rows for WGS_PE_18.2G T=32 and
WGS_PE_40G T=32 that are Amdahl-fit extrapolations.  These are NOT present in
the canonical benchmark CSV (no actual T=32 runs for those datasets) and are
therefore absent from this CSV.  They appear only in the paper table.

**Interpretation:**  
Combined effect of all optimisations: speculative statistics (§5.3), parallel
output compression (§5.4), and enlarged I/O buffers.  Peak measured speedup
4.01× at WGS_PE_40G T=16.

## table8_gpu_overhead.csv

**Paper section:** 8.2  
**Caption:** GPU overhead vs. CPU: absolute overhead (GPU_s − CPU_s) and
relative overhead (%) by dataset and thread count.

**Source file:**  
`docs/publication/figures/fig4_transfer_overhead.csv`

**Extraction:**  
Direct copy; sign-format the overhead columns (+/−).

**How fig4_transfer_overhead.csv was produced:**  
Derived from `full_benchmark_20260417.csv` by pivoting opengene/cpu/gpu wall
times and computing `gpu_overhead_vs_cpu_s = d0bromir_gpu_s − d0bromir_cpu_s`
and `gpu_overhead_vs_cpu_pct = overhead_s / d0bromir_cpu_s × 100`.  Negative
values indicate the GPU path is faster than CPU.
